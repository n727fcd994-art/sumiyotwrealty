# 日本房地產銷售型網站 - 規劃文檔

## 一、網站頁面結構（Sitemap）

```
首頁 (index.html)
├── 物件列表頁 (properties.html)
│   └── 物件詳情頁 (property-detail.html)
├── 關於我們 (about.html)
├── 服務流程 (services.html)
├── 常見問題 (faq.html)
└── 聯絡我們 (contact.html)
```

## 二、每個頁面的目的與重點內容

### 1. 首頁 (index.html)
**目的**：建立第一印象，快速傳達價值主張，引導使用者進入轉換流程

**重點內容**：
- Hero Section：主視覺 + 核心價值主張（「找到您理想的日本不動產」）
- 精選物件區塊：展示 3-6 個熱門物件（卡片式設計）
- 服務優勢：為什麼選擇我們（專業團隊、在地服務、完整售後）
- 成功案例/客戶見證：建立信任感
- 多個 CTA 按鈕：引導加入 Line

### 2. 物件列表頁 (properties.html)
**目的**：展示所有可售物件，提供篩選功能，增加瀏覽深度

**重點內容**：
- 篩選器：地區、價格、類型、坪數
- 物件卡片網格：圖片、標題、價格、地點、基本資訊
- 排序功能：價格、最新、熱門
- 每個物件卡片都有「立即諮詢」CTA
- 頁面底部固定 CTA 條

### 3. 物件詳情頁 (property-detail.html)
**目的**：詳細展示物件資訊，建立購買意願，促成聯絡

**重點內容**：
- 物件圖片輪播（多張照片）
- 物件基本資訊：價格、地址、坪數、建物類型
- 詳細規格：房間數、建成年份、交通資訊
- 周邊環境介紹
- 投資分析（如適用）
- 多個 CTA：頁面頂部、資訊區塊後、頁面底部
- 相關物件推薦

### 4. 關於我們 (about.html)
**目的**：建立品牌信任，展示專業度與在地優勢

**重點內容**：
- 公司/團隊介紹
- 服務理念與價值觀
- 專業證照與認證
- 在地優勢（日本當地團隊）
- 服務流程簡介
- CTA：加入 Line 了解更多

### 5. 服務流程 (services.html)
**目的**：降低購買疑慮，說明服務流程，建立信心

**重點內容**：
- 購買流程圖（步驟式說明）
- 各階段服務內容
- 售後服務說明
- 常見問題預覽
- CTA：開始您的投資之旅

### 6. 常見問題 (faq.html)
**目的**：解答疑慮，減少客服負擔，提升轉換率

**重點內容**：
- 分類式 FAQ（購買流程、稅務、貸款、管理）
- 可展開/收合的問答區塊
- 每個問題後可引導至 Line 諮詢
- 頁面底部 CTA

### 7. 聯絡我們 (contact.html)
**目的**：提供多種聯絡方式，最終引導至 Line

**重點內容**：
- 聯絡方式：Line、電話、Email
- Line QR Code 大圖展示
- 聯絡表單（可選，但主要引導至 Line）
- 營業時間與地址
- 地圖嵌入（如適用）

## 三、CTA 在各頁的配置建議

### CTA 類型
1. **主要 CTA**：加入 Line 聯絡業務（綠色，突出顯示）
2. **次要 CTA**：了解更多、查看物件、預約看屋

### 各頁面 CTA 配置

#### 首頁
- **Hero Section**：大型主要 CTA（「免費諮詢日本不動產」）
- **精選物件區塊**：每個物件卡片有「立即諮詢」按鈕
- **服務優勢後**：中型 CTA（「開始您的投資之旅」）
- **客戶見證後**：CTA（「加入 Line 獲得專業建議」）
- **頁面底部**：固定浮動 CTA 條（始終可見）

#### 物件列表頁
- **每個物件卡片**：小型 CTA（「諮詢此物件」）
- **篩選器下方**：提示文字 + CTA（「找不到理想物件？讓我們幫您」）
- **頁面底部**：固定浮動 CTA 條

#### 物件詳情頁
- **頁面頂部**：固定浮動 CTA（快速聯絡）
- **物件資訊區塊後**：大型 CTA（「立即預約看屋/諮詢」）
- **投資分析後**：CTA（「獲得完整投資報告」）
- **相關物件區塊前**：CTA（「有其他需求？聯絡我們」）
- **頁面底部**：固定浮動 CTA 條

#### 關於我們
- **團隊介紹後**：CTA（「與專業團隊聊聊」）
- **頁面底部**：固定浮動 CTA 條

#### 服務流程
- **每個流程步驟**：小型 CTA（「想了解更多？」）
- **流程圖結束後**：大型 CTA（「開始您的投資之旅」）
- **頁面底部**：固定浮動 CTA 條

#### 常見問題
- **每個 FAQ 區塊**：可選的小型 CTA（「仍有疑問？聯絡我們」）
- **FAQ 結束後**：大型 CTA（「加入 Line 獲得專業解答」）
- **頁面底部**：固定浮動 CTA 條

#### 聯絡我們
- **頁面中央**：大型 Line QR Code + CTA
- **聯絡表單提交後**：引導至 Line
- **頁面底部**：固定浮動 CTA 條

### CTA 設計原則
- **顏色**：Line 綠色 (#00C300) 或品牌主色
- **文字**：行動導向（「立即諮詢」「免費評估」「加入 Line」）
- **位置**：關鍵決策點、資訊閱讀後、頁面可見區域
- **頻率**：每頁至少 2-3 個，但不過度干擾閱讀

## 四、後台系統架構

### 配置檔案結構
使用 JSON 配置檔案管理所有可變更內容：
- `config/site-config.json`：網站基本設定
- `config/properties.json`：物件資料
- `config/content.json`：頁面文案內容
- `config/images.json`：圖片路徑配置

### 後台管理介面
- 簡單的 HTML 表單介面（admin.html）
- 可編輯文字、上傳圖片、管理物件
- 資料儲存在 JSON 檔案中
- 無需技術背景即可維護

### 維護項目
- 物件資訊（新增/編輯/刪除）
- 頁面文案（標題、描述、按鈕文字）
- 圖片更換（上傳新圖片，更新路徑）
- 聯絡資訊（Line ID、電話、Email）
- 網站基本設定（公司名稱、Logo、顏色）
